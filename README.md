一、项目概述
 
1.1 项目定位
改系统是一套面向 IT 教育培训机构的 AI 原生教学辅助系统，以国产大模型（DeepSeek-V3 / DeepSeek-Coder-V2）为驱动核心，基于 LangChain + LangGraph + FastAPI 构建，前端采用 Vue3 + Element Plus 实现。系统服务三类核心学员群体：在校大学生、应届毕业生、职场转行人员，覆盖学员学习全旅程的四大核心教学场景，通过多智能体协同实现教学全流程智能化升级。
 
1.2 核心目标
 
- 为学员提供 7×24 小时智能化学习辅助，提升学习效率与求职竞争力

- 为教师提供自动化教学工具，大幅降低重复性工作负担

- 为机构搭建全流程 AI 教学体系，实现教学质量标准化与可量化

- 输出企业级 AI 智能体全流程开发能力，满足 IT 行业 AI 实战人才需求

三、四大核心 Agent 设计
 
EduAgent 由 4 个专业智能体协同工作，每个 Agent 聚焦一个核心教学场景，各司其职、相互协作，同时支持多 Agent 串联 Pipeline 处理复杂需求。
 
3.1 智能问答 Agent：7×24 小时专属助教
 
核心定位：基于完整 RAG（检索增强生成）链路的精准问答系统，替代传统 AI 闲聊，为学员提供课程知识专属答疑。
 
核心能力设计
 
1. 三级 Query 理解

- 明确问题：直接检索知识库精准匹配

- 模糊问题：通过 HyDE（假设文档嵌入）策略扩展语义，提升召回率

- 宽泛问题：自动改写成 3-5 个子问题并行检索，覆盖完整知识维度

2. 混合召回机制

- 向量语义检索（Milvus）+ 关键词检索（BM25）双路并行，召回 Top-20 候选结果

- 解决单一检索方式的语义偏差与关键词遗漏问题

3. BGE-Reranker 精排

- 用业界领先的重排模型对候选结果二次精排，最终取 Top-3 送入大模型生成

- 大幅提升答案精准度，减少冗余信息干扰

4. 置信度感知回答

- 高置信度答案：附带参考来源标注，可追溯知识出处

- 低置信度答案：自动标注「仅供参考」，并推送至知识库待补充队列

5. 多轮对话记忆

- 支持 10 轮滑动窗口上下文保持，配合摘要压缩机制

- 连续追问不丢失上下文，实现连贯对话体验
 
 
 
3.2 试卷批改 Agent：AI 预批改，教师来把关
 
核心定位：坚持「AI 不替代教师，只帮教师提效」原则，采用「三轨并行」架构，自动化处理全题型批改，实现人机协作。
 
核心能力设计
 
1. 三轨并行架构（异步并发执行）
表格
   
轨道 题型 技术方案 响应速度 
第一轨 客观题（单选/多选/判断） 规则引擎精准匹配 毫秒级 
第二轨 简答题/论述题 LLM 语义评分 + Function Calling 秒级 
第三轨 代码题 Judge0 沙箱 + DeepSeek-Coder-V2 秒-分钟级 
 

- 三轨通过  asyncio.gather  并行执行，大幅提升批改效率

2. 知识薄弱点分析

- 批改完成后自动分析学员错题分布，定位知识薄弱环节

- 生成个性化错题报告与知识点巩固建议

3. 人机协作闭环

- AI 预批改结果需教师登录确认后，才正式发布给学员

- 教师可人工修正评分、补充评语，保障批改准确性
 
 
 
3.3 简历审查 Agent：六维度精准诊断
 
核心定位：针对求职场景，从专业维度深度解析简历，提供可落地的修改建议，提升学员简历竞争力。
 
核心能力设计
 
1. 六维度深度评审

- 工作经历：岗位匹配度、职责描述完整性

- 技能匹配：与目标岗位的技能契合度、技能深度

- 项目描述：技术栈合理性、项目价值呈现

- 量化数据：成果量化程度、数据说服力

- 格式排版：简历结构、视觉规范、可读性

- 表达规范：语言逻辑、专业术语准确性

2. 并行评审机制

- 六个维度分三组并行评审，提升处理效率

- 系统不仅给出评分，更提供带原文定位的具体修改建议，明确标注问题位置与优化方案

3. 三级优先级分类

- 按「高/中/低」优先级呈现修改建议，优先解决核心问题

4. 可视化能力雷达图

- 最终生成直观的能力雷达图，让学员一眼看清简历短板与优势

- 支持对比目标岗位要求，明确提升方向
 
 
 
3.4 模拟面试 Agent：真实面试场景模拟
 
核心定位：完整复现真实技术面试流程，针对学员薄弱点动态出题，提供专业面试评估与改进方案。
 
核心能力设计
 
1. 四阶段状态机设计
遵循真实面试节奏，完整流程： INTRO（自我介绍） → TECH（技术提问） → PROJECT（项目深挖） → REPORT（面试报告） 

2. 动态题目生成

- 根据学员简历、历史作答记录，针对薄弱点优先出题

- 自动去重，避免重复提问，模拟真实面试的递进式考察

3. 双轨并行评估

- 技术深度维度：考察知识点掌握程度、问题解决能力

- 表达能力维度：考察逻辑清晰度、沟通表达能力

- 两个维度互不干扰，全面评估面试表现

4. 结构化报告异步生成

- 面试结束后 30-60 秒内生成完整面试报告

- 包含能力雷达图、失分点分析、针对性改进建议

- 支持多次面试对比，跟踪能力提升轨迹

4.3 多 Agent 串联 Pipeline 设计
 
EduAgent 最核心的特色能力：当学员提出复杂意图时，系统自动识别复杂度分值，触发多 Agent 串联模式，多个 Agent 协同完成端到端的复合任务。
 
场景一：求职全流程辅导
 
- 学员输入：「帮我准备 Java 后端面试，帮我全面辅导一下」

- 系统识别：复合意图（复杂度 4 分），自动触发串联 Pipeline

- 执行流程：

1. 简历审查 Agent：解析简历，输出技能标签与薄弱点

2. 模拟面试 Agent（结构化）：针对薄弱点优先出题，完成面试

3. 聚合节点：整合简历分析与面试报告

4. 输出结果：简历修改建议 + 面试题单 + 能力雷达图
 
场景二：代码作业深度辅导
 
- 学员输入：「帮我分析问题并讲解」（提交代码作业）

- 系统识别：复合意图（复杂度 3 分），触发串联 Pipeline

- 执行流程：

1. 试卷批改 Agent：Judge0 运行 + 代码分析，输出错误类型

2. 智能问答 Agent：检索错误相关知识点，生成详细讲解

3. 聚合节点：整合批改报告与知识讲解

4. 输出结果：批改报告 + 知识点精讲 + 优化代码示例


五、开发推进范围确认

5.1 推进策略（前后端分离）

- 采用前后端分离推进模式：前端与后端分阶段建设、按接口契约对齐联调

- 前端骨架前置：先完成 Vue3 工程壳层、路由、状态管理、通用布局和基础组件规范

- 后端能力优先：前端骨架完成后，以后端核心能力建设为主线，按 Agent 架构顺序逐步落地

5.2 执行原则

- 先壳层、后能力、再联调：先保证前端可运行骨架，再推进后端能力，再做端到端打通

- 每阶段独立验收：阶段目标完成后再进入下一阶段，不跨阶段并行实现核心能力

- 遵循分层约束：API 层仅处理协议，Orchestrator 仅做调度，Agent 通过 Tool 调用能力
